# Contributing

Thanks for considering contributing! Short guide:

1. Fork the repo and create a branch: `git checkout -b feature/your-change`
2. Run tests and linters: `uv run pytest` and `ruff check src tests`
3. Commit logically-scoped changes with clear messages
4. Push and open a PR against `main`

PR checklist:
- Include tests for new behavior
- Update docs if public APIs or behavior changed
- Keep commits focused and atomic
